# Материал для раздела «Тестирование»

## 1. Цель тестирования

Целью тестирования является подтверждение корректности модели графа, алгоритмов
обхода, поиска кратчайших путей, построения минимального остовного дерева и
сохранения данных в JSON-файлы.

Тесты написаны с использованием библиотеки Pytest и располагаются в каталоге
`tests`. Для запуска используется команда `pytest`.

## 2. Проверяемые компоненты

| Компонент | Проверяемые случаи |
|---|---|
| Модель графа | добавление и удаление вершин и рёбер, ориентированный и неориентированный режимы, запрет петель, сериализация |
| BFS | порядок обхода по уровням, родители вершин, расстояние в количестве рёбер, несвязный граф |
| DFS | углублённый порядок обхода, дерево предков, несвязный граф |
| Дейкстра | кратчайшие расстояния, обновление родителей, недостижимые вершины, отрицательные веса |
| Краскал | число рёбер в остове, минимальный суммарный вес, исключение циклов, запрет запуска для ориентированного графа |
| JSON | сохранение структуры графа и координат вершин, восстановление типа графа |

## 3. Примеры тест-кейсов

### Тест 1. BFS на связном графе

**Входные данные:** граф с вершинами A, B, C, D, E и стартовой вершиной A.  
**Ожидаемый результат:** порядок обхода A, B, C, D, E.  
**Фактический результат:** соответствует ожидаемому.  
**Статус:** пройден.

### Тест 2. Дейкстра с несколькими путями

**Входные данные:** граф, в котором до вершины B можно пройти напрямую с
весом 4 или через C с весами 1 и 2.  
**Ожидаемый результат:** кратчайшее расстояние от A до B равно 3, родителем B
является C.  
**Фактический результат:** соответствует ожидаемому.  
**Статус:** пройден.

### Тест 3. Отрицательный вес для алгоритма Дейкстры

**Входные данные:** граф с ребром веса −1.  
**Ожидаемый результат:** программа отказывает в запуске алгоритма и выводит
объяснение причины.  
**Фактический результат:** соответствует ожидаемому.  
**Статус:** пройден.

### Тест 4. Краскал и цикл

**Входные данные:** связный граф, содержащий цикл.  
**Ожидаемый результат:** ребро, образующее цикл, не включается в минимальное
остовное дерево.  
**Фактический результат:** соответствует ожидаемому.  
**Статус:** пройден.

## 4. Вывод

Автоматические тесты подтверждают корректность основных операций с графом и
реализаций алгоритмов. Отдельно проверяются пограничные ситуации: пустые или
несвязные графы, отсутствующая стартовая вершина, отрицательные веса и
попытка применения алгоритма Краскала к ориентированному графу.
