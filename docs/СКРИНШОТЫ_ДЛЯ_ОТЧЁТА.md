# Скриншоты для отчёта: что сделать после запуска программы

В отчёт лучше вставлять реальные скриншоты окна, сделанные на твоём компьютере
после запуска финальной версии. Так они точно будут совпадать с приложением.

## Нужные 4 скриншота

1. **Главное окно с примером графа.**
   Покажи вершины A–E и все рёбра. Подпись: «Рисунок 1 — Главное окно
   приложения с загруженным демонстрационным графом».

2. **Редактирование графа.**
   Добавь вершину F и одно ребро, сделай снимок. Подпись: «Рисунок 2 —
   Добавление вершины и ребра в редакторе графа».

3. **Пошаговый BFS или DFS.**
   Запусти BFS от A, нажми кнопку «▶» несколько раз, чтобы были видны
   подсвеченные вершины и пояснение шага. Подпись: «Рисунок 3 —
   Пошаговая визуализация обхода в ширину».

4. **Алгоритм Дейкстры либо Краскала.**
   Запусти один из них и сделай снимок с текстовым результатом. Для Краскала
   должны быть видны зелёные рёбра минимального остовного дерева. Подпись:
   «Рисунок 4 — Результат работы алгоритма Краскала».

## Как сделать снимок

На Windows нажми `Win + Shift + S`, выдели окно программы, затем вставь
изображение в отчёт через `Ctrl + V`. Лучше сделать снимки без лишних окон
на заднем плане.

## В архиве уже есть два примера

- `пример_интерфейса_финальная_версия.png` — главное окно с демонстрационным графом;
- `пример_BFS_пошаговая_визуализация.png` — окно во время пошагового выполнения BFS.

Это настоящие снимки финальной версии программы, полученные при её запуске. Для финального отчёта лучше также сделать свои снимки на Windows: тогда в кадре будет твой экран и нормальный размер окна.
